(SQL Server) Microsoft SQL Server Error 18456

By JerryWu 08 September 2020 S q l server

本篇同步發佈在 Medium 平台上 Medium 連結

第一次安裝完 Microsoft SQL Server 登入時可能會遇到 Error 18456 的問題

發生原因

使用 SQL Server 驗證，因沒有設定使用 SQL Server 驗證登入，進而產生(Microsoft SQL Server, 錯誤: 18456)

解決方法

  1. 先使用 Window 驗證進行登入

  1. 對資料庫點擊右鍵並選擇屬性

  1. 選擇安全性 → 伺服器驗證區塊勾選 SQL Server 及 Windows 驗證模式(S) → 確定

  1. 會跳出此提示訊息，必須重啟 SQL Server Service 後上述的設定才會生效

  1. 在重啟服務之前我們先確認 SQL Server 登入帳號是否有啟用，安全性 → 找到要登入的帳號名稱 → 點擊右鍵選擇[屬性]

  1. 選擇狀態 → 登入區塊勾選已啟用(E) → 確定

  1. 對資料庫點擊右鍵並選擇重新啟動

  1. 點選[是]，便會開始重啟 SQL Server Service

  1. 完成以上流程就能使用 SQL Server 驗證登入啦!
